📊  Executive Summary
Lumbago Music AI to innowacyjna aplikacja łącząca klasyczne narzędzia DJ-skie z zaawansowaną sztuczną
inteligencją, stworzona dla profesjonalnych DJ-ów, kolekcjonerów muzyki i producentów.
Kluczowe wartości
🎯  Problem
DJ-e tracą godziny na ręczne tagowanie,
porządkowanie i konwersję bibliotek muzycznych
między różnymi platformami.
💡  Rozwiązanie
Inteligentny system automatyzujący 80% pracy
nad biblioteką, z precyzyjną konwersją baz
Rekordbox/VirtualDJ i rozpoznawaniem utworów.
🎪  Rynek docelowy
15 milionów DJ-ów na świecie, 3 mln aktywnych
użytkowników Rekordbox/VirtualDJ.
💰  Model biznesowy
Freemium: wersja podstawowa darmowa, Pro (49
PLN/msc), Enterprise (API dla klubów).
Unikalne cechy (USP)
✓  Jedyna aplikacja z pełną konwersją Rekordbox ↔ VirtualDJ (zachowanie cue points z dokładnością do ms)
✓  AI rozpoznające utwory z 10-sekundowego fragmentu (acoustic fingerprinting)
✓  Inteligentne sugesti kolejności tracków w secie (harmonic mixing + energy flow)
✓  Automatyczne wykrywanie i rozwiązywanie duplikatów (3 metody: hash, tags, audio fingerprint)
🚀  Wprowadzenie i wizja projektu
Kontekst
Nowoczesne narzędzia DJ-skie (Rekordbox, VirtualDJ, Traktor) działają w izolowanych ekosystemach. Przejście
między platformami wiąże się z utratą metadanych, cue points i struktury biblioteki. Ręczne tagowanie i
organizacja tysiąca utworów zajmuje setki godzin.

Wizja
Lumbago Music AI to "szwajcarski scyzoryk" dla DJ-a — jedno narzędzie do wszystkiego:
Automatyczne tagowanie i rozpoznawanie utworów
Konwersja baz między platformami bez utraty danych
Inteligentne sugestie tracków i kolejności w setach
Wykrywanie duplikatów i czyszczenie biblioteki
Backup i synchronizacja z chmurą
Analiza quality mixów i transition scoring
Technologia
Projekt wykorzystuje najnowsze technologie:
Desktop: Python 3.11+, PyQt6, SQLite, Mutagen, lxml
Mobile: Kotlin, Jetpack Compose (Android first)
AI/ML: PyTorch, librosa, chromaprint (AcoustID), scikit-learn
Cloud: AWS S3 / Google Drive API, OAuth 2.0
Performance: Opcjonalne mikroserwisy w Rust dla heavy computations

1. User selects folder(s) + options
2. Worker thread starts: os.walk() for recursive scan
3. For each file:
detect_format() → check extension + magic bytes
read_tags() → Mutagen.File(path)

Proces 2: XML Conversion (Rekordbox → VirtualDJ)
XML Converter
Source: rekordbox_library.xml (Rekordbox 6.x)
Target: virtualdj_database.xml (VirtualDJ 2024)
Mapping:
Title → Title ✓
Artist → Artist ✓
Tempo (BPM) → Bpm ✓
Tonality → Key ✓
POSITION_MARK → Poi (Cue) ⚠  Convert ms
Convert  Dry Run
Backend Flow:
1. Parse source XML: etree.parse(source_path)
2. Extract tracks: for track in root.findall('.//TRACK')
3. Map fields to internal Track model:
class Track: title: str artist: str bpm: float key: str cues:
List[CuePoint] # time_ms, type, label loops: List[Loop] # in_ms, out_ms
4. Convert cues: Rekordbox uses float seconds → VirtualDJ uses int milliseconds
5. Build target XML: construct new tree with VirtualDJ schema
6. Validate: check required fields, warn on missing data
7. Write output + backup original + save conversion log
Proces 3: Duplicate Detection
Duplicate Finder
Mode: Audio Fingerprint (most accurate)

🗄  Baza danych (ERD i schema)
Schemat bazy SQLite
-- Tracks table (główna tabela utworów) CREATE TABLE tracks ( id INTEGER
PRIMARY KEY AUTOINCREMENT, path TEXT NOT NULL UNIQUE, title TEXT, artist TEXT,
album TEXT, genre TEXT, bpm REAL, key TEXT, duration INTEGER, -- w sekundach
file_size INTEGER, -- w bajtach format TEXT, -- mp3, flac, m4a, etc. bitrate
INTEGER, sample_rate INTEGER, date_added DATETIME DEFAULT CURRENT_TIMESTAMP,
date_modified DATETIME, play_count INTEGER DEFAULT 0, rating INTEGER DEFAULT
0, -- 0-5 stars energy REAL, -- 0-100 mood TEXT, fingerprint TEXT, -- acoustic
fingerprint hash waveform_path TEXT, -- path to cached waveform image
artwork_path TEXT, -- path to cached artwork CONSTRAINT valid_rating CHECK
(rating >= 0 AND rating <= 5) ); CREATE INDEX idx_tracks_artist ON
tracks(artist); CREATE INDEX idx_tracks_genre ON tracks(genre); CREATE INDEX
idx_tracks_bpm ON tracks(bpm); CREATE INDEX idx_tracks_key ON tracks(key);
CREATE INDEX idx_tracks_path ON tracks(path); -- Playlists table CREATE TABLE
playlists ( id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT NOT NULL,
description TEXT, created_at DATETIME DEFAULT CURRENT_TIMESTAMP, modified_at
DATETIME DEFAULT CURRENT_TIMESTAMP, is_smart BOOLEAN DEFAULT 0, -- smart
collection flag rules TEXT -- JSON rules for smart collections ); -- Playlist
tracks (many-to-many) CREATE TABLE playlist_tracks ( playlist_id INTEGER NOT
NULL, track_id INTEGER NOT NULL, position INTEGER NOT NULL, added_at DATETIME
DEFAULT CURRENT_TIMESTAMP, PRIMARY KEY (playlist_id, track_id), FOREIGN KEY
(playlist_id) REFERENCES playlists(id) ON DELETE CASCADE, FOREIGN KEY
(track_id) REFERENCES tracks(id) ON DELETE CASCADE ); CREATE INDEX
idx_playlist_tracks_position ON playlist_tracks(playlist_id, position); -- Cue
points table CREATE TABLE cues ( id INTEGER PRIMARY KEY AUTOINCREMENT,
track_id INTEGER NOT NULL, time_ms INTEGER NOT NULL, -- position in
milliseconds type TEXT NOT NULL, -- 'cue', 'loop_in', 'loop_out', 'memory'
label TEXT, color TEXT, -- hex color code FOREIGN KEY (track_id) REFERENCES
tracks(id) ON DELETE CASCADE ); CREATE INDEX idx_cues_track ON cues(track_id);
-- Tags table (user-defined tags) CREATE TABLE tags ( id INTEGER PRIMARY KEY
AUTOINCREMENT, track_id INTEGER NOT NULL, tag TEXT NOT NULL, source TEXT
DEFAULT 'user', -- 'user', 'ai', 'discogs', etc. confidence REAL, -- for AI
tags created_at DATETIME DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (track_id)
REFERENCES tracks(id) ON DELETE CASCADE ); CREATE INDEX idx_tags_track ON
tags(track_id); CREATE INDEX idx_tags_tag ON tags(tag); -- XML mappings (for
conversion tracking) CREATE TABLE xml_mappings ( id INTEGER PRIMARY KEY
AUTOINCREMENT, track_id INTEGER NOT NULL, source_format TEXT NOT NULL, --

List[Track]: """Parse Rekordbox XML and extract tracks""" tree =
etree.parse(xml_path) root = tree.getroot() tracks = [] for track_el in
root.findall('.//TRACK'): title = track_el.get('Name') or '' artist =
track_el.get('Artist') or '' album = track_el.get('Album') or '' # BPM and Key
bpm = track_el.get('AverageBpm') bpm = float(bpm) if bpm else None tonality =
track_el.get('Tonality') key = tonality if tonality else None # Cue points
cues = [] for cue in track_el.findall('.//POSITION_MARK'): cue_type =
cue.get('Type') start = cue.get('Start') if start: cues.append({ 'type':
cue_type, 'time_ms': int(float(start) * 1000), 'name': cue.get('Name', '') })
track = Track( title=title, artist=artist, album=album, bpm=bpm, key=key,
cues=cues ) tracks.append(track) return tracks # Usage tracks =

📚  Słownik pojęć (PL/EN)
Terminy DJ-skie i muzyczne
BPM (Beats Per Minute)
PL: Uderzenia na minutę | EN: Beats Per Minute
Tempo utworu wyrażone w liczbie uderzeń (beatów) na minutę. Np. 126 BPM = 126 uderzeń/minutę.
Beatgrid
PL: Siatka rytmiczna | EN: Beat grid
Mapa uderzeń w utworze, używana do synchronizacji i automatycznego dopasowania tempa.
Cue Point
PL: Punkt startowy/znacznik | EN: Cue point
Zaznaczony moment w tracku, do którego DJ może szybko wrócić. Hot Cue = punkt z przypisanym
klawiszem.
Harmonic Mixing / Camelot Wheel
PL: Miksowanie harmoniczne | EN: Harmonic mixing
Technika łączenia utworów w zgodnych tonacjach (np. 8A → 9A). Camelot Wheel to system notacji kluczy
muzycznych (1A-12B).
Loop
PL: Pętla | EN: Loop
Zapętlony fragment utworu (np. 4, 8, 16 taktów), odtwarzający się w kółko.
Pitch / Tempo
PL: Wysokość tonu / Tempo | EN: Pitch / Tempo
Pitch = wysokość dźwięku. Pitch slider zmienia tempo utworu. Key Lock = zachowanie tonacji przy
zmianie tempa.

Waveform
PL: Przebieg fali / Fala dźwiękowa | EN: Waveform
Wizualizacja amplitudy dźwięku w czasie. Pozwala na wizualną identyfikację struktur utworu (drop,
breakdown).
Terminy techniczne
Audio Fingerprint
PL: Odcisk akustyczny | EN: Audio fingerprint
Unikalna sygnatura dźwiękowa utworu, używana do identyfikacji (np. Shazam). Niezależna od formatu i
jakości.
Tag / Metadata
PL: Etykieta / Metadane | EN: Tag / Metadata
Informacje o pliku audio zapisane w nagłówku (ID3 dla MP3): tytuł, artysta, album, rok, okładka, etc.
Bitrate
PL: Przepływność bitowa | EN: Bitrate
Jakość audio wyrażona w kbps (kilobitów na sekundę). 320 kbps = najwyższa jakość MP3, 1411 kbps =
FLAC.
Sample Rate
PL: Częstotliwość próbkowania | EN: Sample rate
Liczba próbek audio na sekundę (Hz). Standard: 44100 Hz (CD quality), 48000 Hz (studio).
FLAC
PL/EN: Free Lossless Audio Codec
Format audio bez strat jakości. Większe pliki niż MP3, ale zachowuje 100% oryginalnego dźwięku.
XML
PL/EN: Extensible Markup Language
Format pliku tekstowego do przechowywania danych. Rekordbox i VirtualDJ używają XML do eksportu
bibliotek.
SQLite
PL/EN: SQLite Database
